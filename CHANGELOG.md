# 小耳朵 更新说明

## v0.2.0 — 声音底纹通用化 + Bug修复

### 新功能：声音底纹（Voice Baseline）

小耳朵可以学习主人正常说话的声音特征，之后每次收到语音时自动对比"和平时有什么不同"。

**怎么用：**

1. 发几条正常说话的语音（建议 5 条以上，越多越稳定）
2. 每发完一条，发送指令：`收声音样本`
3. 之后再收到语音时，小耳朵会自动在系统提示里附上声纹变化信息

**声纹变化长什么样：**

```
声纹变化：音高↓ | 响度↑
```

只有和平时相比有明显变化的维度才会出现。没变化 = 不输出任何东西。

主模型收到这行信息后，结合自己听到的语音内容就能判断说话人的状态（低落、兴奋、唱歌、困了等等），不需要小耳朵替它下情绪标签。

**可检测的维度：**

| 符号 | 含义 |
|------|------|
| 音高↑/↓ | 说话音调比平时高/低 |
| 响度↑/↓ | 说话声音比平时大/轻 |
| 语速↑/↓ | 说话节奏比平时快/慢 |
| 停顿↑/↓ | 停顿次数比平时多/少 |
| 亮度↑/↓ | 声音频谱比平时亮（尖锐）/闷 |
| 音高走势↗/↘ | 这段话后半截比前半截高/低 |
| 响度走势↗/↘ | 这段话后半截比前半截响/轻 |

**相关指令：**

| 指令 | 作用 |
|------|------|
| `收声音样本` | 把最近一条语音收为底纹样本 |
| `声音底纹` | 查看当前底纹状态 |
| `声音对比` | 手动对比最近一条语音和底纹 |

### 默认模式改为自动模式

`voice_api_tier` 配置项默认值从 `legacy` 改为 `auto`。自动模式的逻辑：
- 检测到主模型支持音频输入 → 直接把语音挂给主模型听 + 附加声纹变化提示
- 不支持 → 走小耳朵报告模式转写

已有用户如果在配置里手动写了档位，不受影响。

### Bug 修复

1. **内存泄漏修复** — 直连音频缓存（PENDING_DIRECT_AUDIO）增加定期清理机制，每小时自动回收超过 180 秒未使用的条目
2. **时序错位修复** — 防止前一条语音的音频被误挂到后续纯文字消息上。新增 10 秒时序保护窗口：如果当前消息本身不含音频组件且距离上次语音已超过 10 秒，不挂载

### 安装

不需要安装新依赖。所有使用的库和之前版本一样。

### 从旧版升级

直接用新插件文件夹替换旧版即可。如果之前用过声音底纹功能，旧的样本文件仍然有效，插件会自动读取。
